# Experiments

## Методика

Эксперименты выполняются командой:

```bash
make experiment
make verify-results
```

Скрипт использует фиксированный seed `4116`, сбрасывает состояние стенда через `/admin/reset`, выполняет сценарии в заданном порядке и сохраняет результаты в `results/latest/`.

## Сценарии

| Сценарий | Проверка | Ожидаемый результат |
| --- | --- | --- |
| `sync_transaction` | 80 синхронных команд начисления золота. | p50/p95 и throughput зафиксированы, команды изменили PostgreSQL. |
| `outbox_delivery` | 80 команд с записью Outbox и асинхронной публикацией. | События доставлены worker, pending Outbox отсутствует после ожидания. |
| `duplicate_callback` | 5 одинаковых команд с одним `message_id`. | Баланс изменен один раз, 4 запроса распознаны как дубли. |
| `publish_no_ack` | Worker публикует событие, но не отмечает Outbox доставленным. | Redis Streams содержит транспортный дубль, Outbox в итоге закрыт. |
| `pubsub_loss` | Сообщение публикуется до подписки. | Поздний подписчик его не получает. |
| `cache_stale` | Значение читается из cache-aside, затем PostgreSQL меняется без инвалидации. | Повторное чтение возвращает устаревший кэш. |
| `saga` | Одна успешная и одна компенсированная покупка. | Наблюдаются `saga_completed >= 1` и `saga_compensated >= 1`. |
| `websocket_replay` | WebSocket-клиент подключается с `last=0-0`. | Клиент получает хотя бы одно replay-событие. |

## Проверка результатов

`scripts/verify_results.py` проверяет следующие условия:

- оба нагрузочных батча выполнены полностью;
- latency действительно измерена;
- Inbox предотвратил повторное изменение состояния;
- транспортный дубль после `publish_no_ack` обнаружен;
- Pub/Sub потерял сообщение для позднего подписчика;
- cache-aside вернул stale value;
- saga завершилась и компенсировалась;
- WebSocket replay доставил события;
- после эксперимента нет pending Outbox.

Если хотя бы один инвариант нарушен, команда `make verify-results` завершается с ненулевым кодом.
